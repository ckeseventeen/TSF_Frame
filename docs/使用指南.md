## 3. 数据准备

### DataFrame 期望格式

```python
import pandas as pd

df = pd.DataFrame({
    'date':               pd.date_range('2012-01-31', periods=144, freq='ME'),
    'monthly_deposit':    [...],   # 缴存额
    'monthly_withdrawal': [...],   # 提取额
    'loan_balance':       [...],   # 贷款余额
    'depositor_count':    [...],   # 缴存人数
    # 其它字段按业务添加
})
```

要求:
- `date` 列必须是 `datetime64[ns]`
- 频率为月度(`'ME'` 或 `'MS'`)
- 非负业务量列不允许出现负值(`HPFAdapter.validate_data` 会拒绝)

### 数据存放约定

```
data/
├── raw/        # 原始数据 (gitignore)
├── processed/  # 清洗后的数据 (gitignore)
└── reference/  # 训练期参考(漂移检测基线, gitignore)
```

## 4. 业务适配器(HPFAdapter)

适配器是"业务防腐层":校验、预处理、反变换、业务指标。

### 基本用法

```python
from configs.hpf import HPFConfig
from tsf_frame.business.hpf_adapter import HPFAdapter

cfg = HPFConfig()
adapter = HPFAdapter(cfg.to_adapter_config())   # 注意 .to_adapter_config()

# 1) 校验
ok, msg = adapter.validate_data(df)
assert ok, msg

# 2) 预处理(异常值处理 + 归一化)
processed_df, meta = adapter.preprocess(df)

# 3) 反归一化(预测后)
pred_df = adapter.postprocess(y_pred, meta)

# 4) 业务指标 (MAPE / direction_accuracy / yoy_mae / qoq_mae)
metrics = adapter.get_business_metrics(y_true_df, y_pred_df)
```

### 常用配置

```python
cfg = HPFConfig()
cfg.data.target_columns        = ['monthly_deposit']     # 预测目标
cfg.feature.use_time           = True                    # 启用时间特征
cfg.feature.lags               = [1, 3, 6, 12]           # 滞后阶数
cfg.feature.rolling_windows    = [3, 6, 12]              # 滚动窗口
cfg.model.model_name           = 'xgboost'
cfg.model.probabilistic        = True
cfg.model.probabilistic_method = 'residual'              # residual / mc_dropout / quantile
```

### 政策情景分析(HPF 特有)

```python
adjusted = adapter.get_policy_adjusted_forecast(
    base_forecast=base_df,                        # DataFrame
    policy_effects={'monthly_deposit': 0.05},     # 政策使缴存额增长 5%
)
# 内部用 (1 + effect) 乘性调整, 自动 clip 非负
```

## 5. 特征工程

所有特征工程类遵循 sklearn 的 fit→transform 模式,**严格因果**。

### 单个工程器

```python
from tsf_frame.features.engineering import (
    TimeFeatureEngineer, LagFeatureEngineer,
    RollingFeatureEngineer, DifferenceFeatureEngineer,
)

# target_cols 是列表;省略时自动选所有数值列
lag = LagFeatureEngineer({'target_cols': ['monthly_deposit'], 'lags': [1, 3, 6]})
lag.fit(train_df)
train_feat = lag.transform(train_df)
test_feat  = lag.transform(test_df)            # 用训练期 fit 的参数

# TimeFeatureEngineer 需要 DatetimeIndex 或通过 time_col 指定时间列
time_eng = TimeFeatureEngineer({
    'time_col': 'date',
    'features': ['month', 'quarter', 'year', 'is_weekend']
})
```

### 组合工程器(推荐)

```python
from tsf_frame.features.engineering import create_feature_engineer

feature_config = {
    'time_config':       {'features': ['month', 'quarter', 'year']},
    'lag_config':        {'target_cols': ['monthly_deposit'], 'lags': [1, 3, 6, 12]},
    'rolling_config':    {'target_cols': ['monthly_deposit'],
                          'windows': [3, 6, 12], 'stats': ['mean', 'std']},
    'difference_config': {'target_cols': ['monthly_deposit'], 'periods': [1, 12]},
}

engineer = create_feature_engineer(
    feature_types=['time', 'lag', 'rolling', 'difference'],
    config=feature_config,
)

# time 特征要求 DatetimeIndex 或 time_col 指定的列
df_with_idx = df.set_index('date')
df_feat = engineer.fit_transform(df_with_idx).dropna()

X = df_feat.drop(columns=['monthly_deposit'])
y = df_feat['monthly_deposit']
```

### 特征选择 + 降维

```python
from tsf_frame.features import get_selector, get_reducer

selector = get_selector('kbest', {'k': 20})
selector.fit(X_train, y_train)
X_train_sel = selector.transform(X_train)

reducer = get_reducer('pca', {'n_components': 10})
X_train_pca = reducer.fit_transform(X_train_sel)
```

可选选择器: `kbest` / `rfe` / `model_based` / `lasso` / `correlation` / `variance`
可选降维器: `pca`

### XGBoost 一阶差分技巧

XGBoost 是树模型,无法外推趋势。框架对长期趋势序列默认采用**一阶差分 + cumsum 还原**:

```python
dy_train = y_train.diff().dropna()
model.fit(X_train.iloc[1:], dy_train)
dy_pred = model.predict(X_test)
y_pred = y_train.iloc[-1] + np.cumsum(dy_pred)
```

详见 `pipelines/run_hpf_forecast.py` 的 XGBoost 路径。

# 数据形状: X (N, seq_len, input_size); y (N, output_size)

### 月度稀疏数据提醒

12 年 × 12 月 = 144 条样本对 Transformer 系列**极少**。优先级建议:

1. **首选** XGBoost / LightGBM(强特征 + 一阶差分)
2. **次选** LSTM(小模型,最少 dropout)
3. **慎选** Autoformer / TimesNet(只有在加充足外生变量时)

## 8. 概率预测

三种策略均返回统一的 `ProbabilisticPrediction`:

```python
from tsf_frame.models.base_model import ProbabilisticPrediction

prob: ProbabilisticPrediction = model.predict_probabilistic(X_test)
prob.mean        # 点预测
prob.lower       # 置信下界
prob.upper       # 置信上界
prob.std         # 标准差
prob.samples     # MC 采样原始数据 (S, N, output_size), 仅 mc_dropout
prob.to_dict()   # 可序列化字典
```

| 方法 | 适用模型 | 配置 | 特点 |
|------|---------|------|------|
| `residual` | 所有模型 | `probabilistic_method='residual'` | 用训练残差经验分布,简单稳健;**多输出场景按列分别算分位数**(已修) |
| `mc_dropout` | 深度学习 | 模型含 Dropout 层 | 贝叶斯近似,捕捉模型不确定性 |
| `quantile` | DLinear / 自定义 | `probabilistic_method='quantile'` + `quantiles=[0.025, 0.5, 0.975]` | PinballLoss 训练,精确分位数 |

### 多输出残差区间(2026-Q2 修复)

多输出模型(同时预测多个目标)的残差区间**按列独立计算**,不会被量纲小的维度拖累。原先 `flatten()` 混算的 bug 已修。

## 9. 可视化(PredictionPlotter)

**项目所有图统一从 `PredictionPlotter` 出**。它有 7 个原子方法 + 4 个复合工具。

### 复合工具(一行出图)

```python
from tsf_frame.visualization import PredictionPlotter

plotter = PredictionPlotter(figsize=(14, 8), dpi=120)

# 1) 多模型对比 + 最优模型概率区间 (2x1 子图)
plotter.forecast_comparison_fig(
    x=dates, y_true=y_test,
    models_pred={'XGBoost': pred1, 'LightGBM': pred2},
    best_interval=(pred1, lo, hi),       # (mean, lower, upper) 或 None
    target_label='monthly_deposit',
    save_path='./logs/outputs/foo.png',
)

# 2) 多组指标柱状对比
plotter.metrics_bars_fig(
    groups={
        'MAPE(越低越好)': (models, mape_vals, '{:.2f}%'),
        'R²(越高越好)':   (models, r2_vals,   '{:.4f}'),
    },
    save_path='./logs/outputs/metrics.png',
)

# 3) DL 训练曲线 (1x2: train + val loss)
plotter.training_curves_fig(
    results={name: history_dict for name, history_dict in results.items()},
    log_scale=True,
    save_path='./logs/outputs/curves.png',
)

# 4) 特征重要性 Top-N (横向柱状)
plotter.feature_importance_fig(
    names=feat_names, values=importances, top_n=10,
    save_path='./logs/outputs/feat_imp.png',

```

### 原子方法(自由组合到任意 ax)

```python
import matplotlib.pyplot as plt
fig, axes = plt.subplots(2, 2, figsize=(14, 8))

plotter.lines_compare(axes[0, 0], x=dates,
                      baseline=y_true, series={'pred': y_pred})
plotter.interval_band(axes[0, 1], x=dates,
                      y_pred=mean, y_lower=lo, y_upper=hi, y_actual=y_true)
plotter.bars(axes[1, 0], names=models, values=mapes, fmt='{:.2f}%')
plotter.histogram(axes[1, 1], values=residuals, bins=30)

plotter.save(fig, './logs/outputs/dashboard.png')
```

可用原子方法:
- `lines_compare` — 多 series 折线对比 + 可选 baseline
- `interval_band` — 预测均值 + 概率区间
- `bars` / `bars_h` — 垂直/水平柱状(自动数值标注)
- `histogram` — 直方图
- `scatter_categorical` — 类别散点(告警时间线)
- `residuals` — 残差时序(零参考线)

## 10. 监控 — 快速上手

```python
from tsf_frame.monitoring import (
    ModelMonitor, RuleEngine, SQLiteStore,
    DataDriftDetector, ConsoleChannel, FileChannel,
)

# 装配监控栈
monitor = ModelMonitor(
    model_id='xgb_deposit_v1',
    store=SQLiteStore('./logs/monitor/hpf_monitor.db'),     # 持久化
    data_drift=DataDriftDetector(reference=X_train.values), # 漂移基线
    rule_engine=RuleEngine(                                  # 规则引擎
        rule_ids=['R1_NON_NEGATIVE', 'R2_SUDDEN_CHANGE'],
        params={'R2_SUDDEN_CHANGE': {'max_ratio': 0.30}},
    ),
    window_size=36,        # 队列保留 3 年
    metric_window=12,      # MAPE 算近 12 个月
    cold_start_samples=6,  # 累计 < 6 个月时跳过漂移检测
)
monitor.alert_manager.add_channel(ConsoleChannel())
monitor.alert_manager.add_channel(FileChannel('./logs/monitor/alerts.log'))
monitor.set_performance_baseline({'mape': 0.05, 'mae': 1.0})

# 1) 跑批: 记录预测 (target_ts 作为对齐键)
for ts, x_row, y_pred_val in zip(test_ts, X_test, y_pred):
    monitor.record_prediction(
        timestamp=ts,                # 即 target_ts
        prediction=float(y_pred_val),
        actual=None,                 # 真值还没到, 异步回填
        features=x_row,
    )

# 2) 真值到达后回填 (HPF 月度场景关键!)
for ts, y_true_val in zip(actual_arrival_ts, actuals):
    monitor.settle_actual(target_ts=ts, y_actual=float(y_true_val))

# 3) 周期性检查 + 告警
status = monitor.check_status()
print(status.alert_level)              # 'info' / 'warning' / 'error' / 'critical'
print(status.recommendations)
print(status.needs_retraining)
```

完整示例: `pipelines/examples/hpf_monitoring_example.py`(包含规则注册、漂移检测、报表生成)。

### 异步真值回填(HPF 关键设计)

月度跑批场景下,**预测和真值不同时到达**:
- 跑批日 T 输出 ŷ(T+1) → `record_prediction(timestamp=T+1, prediction=..., actual=None)`
- 一个月后真值到达 → `settle_actual(target_ts=T+1, y_actual=...)`

`PerformanceMonitor` 用 `target_ts` 作为对齐键(不依赖位置索引),即使窗口满+真值乱序到达,也不会错位。

## 11. 监控 — 完整能力清单

### 11.1 6 大子系统(全部可插拔)

| 子系统 | 类 | 作用 |
|--------|------|------|
| 性能监控 | `PerformanceMonitor` | 滑窗 MAE/MSE/RMSE/MAPE/SMAPE/R² + PICP/MIW/Winkler |
| 数据漂移 | `DataDriftDetector` | 特征 PSI / KS,默认 PSI warn=0.1 / crit=0.25 |
| 概念漂移 | `ConceptDriftDetector` | 残差均值偏离 / 方差扩大 / 滚动趋势 |
| 预测漂移 | `PredictionDriftDetector` | y_pred 分布 PSI / KS / JS |
| 数据质量 | `DataQualityMonitor` | 5 个检查器(missing/outlier/schema/freq/range) |
| 业务规则 | `RuleEngine` | 装饰器注册的可插拔规则 |
| 重训触发 | `RetrainingTrigger` | 5 条默认规则(MAPE/漂移/时间/样本量) + 冷却期 |

### 11.2 内置规则

```python
from tsf_frame.monitoring import RuleEngine

eng = RuleEngine(
    rule_ids=[
        'R1_NON_NEGATIVE',       # 非负约束 (CRITICAL)
        'R2_SUDDEN_CHANGE',      # 月环比突变 (默认 30%)
        'R3_OUT_OF_BAND',        # 超出业务值域
        'R4_MONOTONIC_EXPECTED', # 单调约束(累计余额)
    ],
    params={
        'R2_SUDDEN_CHANGE': {'max_ratio': 0.30},
        'R3_OUT_OF_BAND':   {'lo': 0, 'hi': 1000},
    },
)
```

### 11.3 自定义规则(@register_rule)

```python
from tsf_frame.monitoring import register_rule, RuleViolation, AlertLevel

@register_rule('R_HPF_NON_NEGATIVE')
def my_rule(*, prediction, target='monthly_deposit', **_):
    if (prediction < 0).any():
        return [RuleViolation(
            rule_id='R_HPF_NON_NEGATIVE',
            severity=AlertLevel.CRITICAL,
            message=f'{target} 预测出现负值',
        )]
    return []

# 使用
eng = RuleEngine(rule_ids=['R_HPF_NON_NEGATIVE'])
```

## 12. 多目标 / 多步监控

### 12.1 单模型多目标(温度+湿度+气压)

```python
from tsf_frame.monitoring import MultiTargetMonitor

mtm = MultiTargetMonitor(
    model_id='weather',
    targets=['temperature', 'humidity', 'pressure'],
    window_size=200, metric_window=24,
    baseline_per_target={
        'temperature': {'mape': 0.03},   # 温度基线 3%
        'humidity':    {'mape': 0.10},   # 湿度天然不稳, 放宽
        'pressure':    {'mape': 0.005},  # 气压非常稳定
    },
    probabilistic_targets=['temperature'],  # 只温度有区间
)

# 一次记多个目标 (支持只覆盖部分)
mtm.record_prediction(
    timestamp=now,
    predictions={'temperature': 25.3, 'humidity': 60.0, 'pressure': 1013.2},
    y_lowers={'temperature': 24.0},   # 仅温度有区间
    y_uppers={'temperature': 26.5},
)

# 真值部分到达 (湿度晚到)
mtm.settle_actuals(target_ts=now, actuals={'temperature': 25.1, 'pressure': 1013.5})
mtm.settle_actual(target_ts=now, target='humidity', y_actual=58.2)

# 各目标独立 MAPE
for tgt, m in mtm.current().items():
    print(f'{tgt:12s}  MAPE={m["mape"]:.2%}')
```

## 15. 常见问题

**Q: 我能把监控接到 Airflow 里吗?**
A: 可以。`record_prediction` 是幂等的,在任务里循环调用即可。SQLite WAL 模式支持并发读写。

**Q: 重启后内存窗口会丢吗?**
A: 内存 deque 会丢,但 SQLite 三表(predictions / metrics_snapshot / alerts)全部持久化。`PlotReport` 从 SQLite 读,不依赖内存窗口。需要重启后滑窗延续,可在 `__init__` 后从 store 重填。

**Q: 告警怎么推到钉钉/企业微信/邮件?**
A: 用 `CallbackChannel` 包一个 callable,见 §11.4 例子。或者实现一个 `AlertChannel` 子类,用 `@register_alert_channel('dingtalk')` 注册。

**Q: 阈值怎么调?**
A: 全部都在 `HPFMonitoringConfig` / `RuleEngine.params` / 各 detector 的构造参数里,**不需要改代码**。

**Q: HPFMonitor 在哪?**
A: 已废弃。改用通用 `ModelMonitor` + `@register_rule` 注入 HPF 业务规则(见 `pipelines/examples/hpf_monitoring_example.py`)。

**Q: `metric_window` vs `window_size` 区别?**
A: `window_size` 是存储队列容量(保留多少历史 target_ts); `metric_window` 是算指标时只看最近 N 条已 settle 的记录。HPF 月度推荐 `window_size=36, metric_window=12`(保留 3 年,算近 1 年 MAPE)。

**Q: GPU 训练支持吗?**
A: 支持。把 `device='cuda'` 传进 DL 模型 config 即可。所有 DL 模型 `__init__` 已正确把参数搬到目标设备(2026-Q2 修复)。
